Graph¶

All nodes are one of the following types:

Node Types¶

APP
Application of a function or operator to a set of values.
OP
Operator a mapping between multiple operands returning a single result.
VAL
A value providing bytes.

Node Instances¶

The core graph node types in Blaze are:

  • App
  • ArrayNode
  • ExpressionNode
  • FloatNode
  • Fun
  • FunApp
  • IndexNode
  • IntNode
  • Literal
  • Op
  • Slice
  • StringNode

The domain and codomain of an operator are determined only in the context of an application binding.

The domain and codomain of an function are fixed by definition.

Holds the Blaze expression objects.

  • App
  • Op
  • ArrayNode
  • IntNode
  • FloatNode
  • StringNode
  • Assign
  • ExpressionNode
  • Fun
  • FunApp
  • IndexNode

These form the function terms in the ATerm representation of the graph.

class blaze.expr.graph.App(operator)¶

Operator application.

class blaze.expr.graph.ArrayNode(children)¶

A array structure with dimension and length.

T¶

Equivalent to .transpose(), which returns a graph node.

flat¶

Equivalent to .reshape(), which returns a graph node.

class blaze.expr.graph.ExpressionNode(children)¶

A abstract node which supports the full set of PyNumberMethods methods.

class blaze.expr.graph.Fun(arguments)¶

A generic function application. Normally constructed by @lift’ing a function into the Blaze runtime.

class blaze.expr.graph.Op(op, operands)¶

A typed operator taking a set of typed operands.
